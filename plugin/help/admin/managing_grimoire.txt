+grimoire/add <branch>=<name>/<min skill>/<difficulty>/<description>
+grimoire/edit <id>=<name>/<min skill>/<difficulty>/<description>
+grimoire/delete <id>
+grimoire/approve <job id>
+grimoire/reject <job id>=<reason>

Staff commands for managing the Grimoire. These require admin access or the configured management permission.

PERMISSION CONFIGURATION
By default, these commands require the 'manage_grimoire' permission. You can change this to use any existing permission in your game's config/grimoire.yml:

  permissions:
    manage: manage_apps

This allows reuse of existing permissions (manage_apps, manage_jobs, etc.) without plugin modification.

ADDING SPELLS
+grimoire/add creates a new approved spell immediately. The spell is added to the Grimoire and available for casting right away. This bypasses the proposal workflow.

EDITING SPELLS
+grimoire/edit updates an existing spell. You cannot change a spell's branch (must delete and recreate if needed). All four fields (name, min skill, difficulty, description) must be provided even if only changing one.

DELETING SPELLS
+grimoire/delete permanently removes a spell from the Grimoire. This also removes any learned spell records associated with it.

SPELL PROPOSALS
Spell proposals are submitted by players with +grimoire/propose, which creates a standard Ares Job in the configured Grimoire request category. The Job is the source of truth for proposal status.

+grimoire/approve <job id> approves a spell proposal. The spell is created and marked approved, and the Job is closed with an approval comment. If the spell already exists or the proposal was already processed, the command will fail.

+grimoire/reject <job id>=<reason> rejects a spell proposal. The Job is closed with the rejection reason. No spell is created. The player is not notified automatically.

See also: +grimoire
